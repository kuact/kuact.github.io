这是一个鄙视链问题么？额…..好像是的。这前两天，我还在用Ghost做操作系统镜像还原。sigh，不思进取，真是要被piapia打脸。

一直以来用Ghost的唯一功能就是OS镜像还原。Win XP，Win 7再到Win 10。一番狼吞虎咽的全新的OS安装，再用Ghost做一个系统分区镜像。随着时间的推移，当蓝屏，软件不兼容，注册表乱飞的时候，我就时不时把整个OS镜像还原一下，来一个清爽的OS，继续战斗。

Sigh，这么繁琐的操作，我居然都坚持了那么多年，全然不知Win 10自带镜像备份与还原已经那么成熟。

很NB的一点，人家早就实现热备份（VSS Shadow Copy）了，Ghost还得用U盘挂载PE进行冷启动进行离线备份。额，我这是在玩泥巴么。

好吧，我错了；给个机会把….

知识点

Windows Recovery Environment (RE)

Win RE 是WIN10 预装的恢复环境分区（要重启)，主要用于系统重置和镜像恢复（前提是你做了镜像）

  • Win 10 运行状态下进入

选择“开始”->“设置”->“更新和恢复”->“恢复”-> “高级启动”，单击“立即重启”

基本上有了Win RE，事情就成功了一半。你可以通过进入Win RE进行镜像备份与恢复。

Windows Preinstallation Environment (PE)

一个加载在内存里的Win 10微内核；原意是方便企业的大规模电脑部署。个人用它主要是用于离线镜像恢复（之前打脸的ghost）。市面上爆款的微PE就是根据Win PE进行的深度定制。

但是市场上的定制PE很多都带着私货；自己弄一个WIN PE，step by step就可以了。那么问题来了；我有了WIN RE，弄它干嘛？额，万一WIN RE进不去了呢?

还真不用，win 10下直接create a recovery drive直接做USB 启动的WIN RE

System File Checker (SFC)

如果偶尔蓝屏，可以管理员登陆尝试修复一下系统文件： C:\Windows\system32>sfc /scannow

此工具无法修复WIM镜像。

Deployment Image Servicing and Management (DISM)

这是微软自带的命令行，你要对DISM不熟，可以尝试国人开发的DISM+图形化界面呀。总之DISM是一个很NB的工具集。

WIN 10镜像备份

  • SETTING -> Backup -> Go to Backup and Restore (Windows 7)

基本上无脑操作一把，对系统分区（C:）做镜像，镜像文件存（D:）

小提示 如果你使用SoftPerfect RAMdisks，必须把虚拟硬盘设置为”mount as removable”。如果启动了sanbox，也请一并关掉。

WIN 10镜像恢复

  • SETTING -> Recovery -> Advanced Startup

重启，进入Restore Image进行恢复

Win 10 PE制作

Windows ADK（Assessment and Deployment Kit）

制作WinPE 必备，除了这个ADK，你要下载它的Add on（WinPE）。然后管理员运行Deployment and Imaging Tools Environment

  • copype amd64 C:\WinPE_amd64

小提示 将winPE拷贝到C盘的WinPE_amd64 目录

  • MakeWinPEMedia /UFD C:\WinPE_amd64 F: 小提示： 设置正确U盘盘符，这里是F盘（FAT32）

  • MakeWinPEMedia /ISO C:\WinPE_amd64 C:\WinPE_amd64\WinPE_amd64.iso 小提示 制作ISO

diskpart

Windows自带磁盘管理工具，对U盘格式化。

  • list disk 小提示 找到U盘

  • select disk 小提示 一定要选对硬盘/U盘

  • clean 小提示 擦除硬盘

  • create partition primary 小提示 新建主分区（默认分区号1）

  • select partition 1

  • format FS=FAT32

  • assign letter=F 小提示 格式化，分配盘符为F

  • exit

Bootable USB

用Rufus分区选MBR Partition Scheme for BIOS, 文件系统NTFS；ISO文件选WinPE_amd64.iso

笔记本需设置Legacy Bios，以支持U盘启动。现在笔记本/台式机都是用F12快捷键选one time boot manu

如果找不到U盘，尝试冷启动试试？

彩蛋

MBR 与 GPT

3.5寸硬盘出来那么多年，怎么也跑不脱柱面（cylinder），磁头（header），扇区（sector）这三驾马车。真正存储数据的地方在扇区，那么一块硬盘有N多的扇区，人们就发明了LBA（Logical Block Address）做扇区寻址。

MBR：Master Boot Record就是主分区的第一个扇区（LBA0），专门用来引导操作系统启动的。一个扇区一共只有512字节（Byte)，用存储LBA寻址的只有16个字节(Byte)；简单的说，MBR能够寻址到的硬盘扇区加起来只有2TB （2的16次方扇区）。

操作系统夹在文件与扇区的在中间，所以它用了另外一个概念叫文件系统：Windows的FAT,FAT32和NTFS，Linux的exFAT，VMWARE的VFS….

BIOS 与 UEFI

说硬盘的MBR 和 GPT 并非我本意，主要是要配合U盘启动不是。硬盘容量在突飞猛进，放在今天，2TB够个屁呀，于是GPT就来了。准确来说是UEFI带着GPT来强推MBR + BIOS的。

BIOS的出来的年纪大你好几轮，所以它真的老了，很多电脑的新概念它转不动（比如硬盘容量），于是Unified Extensible Firmware Interface (UEFI)就冒了出来；抢眼的是它对新硬件与接口的支持。但BIOS瘦死的骆驼比马大呀，所以UEFI还得继续兼容BIOS一段日子。

来来来，恭喜一下明日之星。

另外不要忘记做WinPE ISO启动盘的时候选在MBR + BIOS。

小提示： BIOS的启动文件叫boot，UEFI的启动文件叫EFI。

cipher 和 format

都是windows自带的

format D: /P:5

随机覆盖5次

WIN10备份总结

  1. WIN10 没有增量备份，只保留一个最新的
  2. WIN10 比Ghost占用更多的备份空间（大约多40%）
  3. WIN10 的备份文件可以直接拷贝
  4. 用WIN10 RE恢复镜像时需要输入windows 密码，如果你用outlook账号，你得启用管理员密码
  5. 用WIN10 RE恢复对NAS的支持不友好（反正我用NAS 路径没法恢复）
